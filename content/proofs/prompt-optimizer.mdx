## Problem

Paste a rough prompt, get a sharper one back — without creating an account, storing your key server-side, or locking into a vendor.

Prompt tooling forces three frictions: account creation, server-side key storage, provider lock-in. Most "optimizer" apps are also black-box rubrics you can't audit or tune.

## Approach

Vite + React 19 PWA. Multi-turn chat UI coaches the prompt live, streaming Gemini tokens through a thin Vercel Edge SSE handler at `/api/chat`. The user's key travels as a Bearer header, never stored server-side. All state (chats, key) persists in `localStorage` — fresh browser = fresh state.

The interactive flow is small:

- Paste a draft prompt.
- Chat with the coach model until the prompt is sharp.
- Edit-and-resubmit on any past user bubble to branch.
- Install as a PWA for offline-first behaviour.

Stack: Vite 7, React 19, strict TypeScript, vite-plugin-pwa, Vercel Edge. No database, no accounts, no server-side key storage.

## Outcome

PR #6 (chat rewrite) merged and deployed at [promptoptim.vercel.app](https://promptoptim.vercel.app). v2 persistence with one-time auto-migration from v1 sentinel-guarded. First-boot seeds a canned demo chat so new visitors can try without pasting a key. Dark-only indigo-violet design system (PR #3/#4) shipped alongside.
