# portfolio-v2 ‚Äî project rules

## Stack (locked ‚Äî do not upgrade without explicit instruction)
- **Next.js** `^15.5.15` ‚Äî App Router only (`app/` dir); no Pages Router
- **React** `^19.0.0`
- **TypeScript** `^5.7.3` ‚Äî strict mode, `moduleResolution: bundler`
- **Tailwind CSS** `^4.0.0` ‚Äî v4 `@theme {}` syntax in `app/globals.css`; **never use v3 `theme.extend` syntax**
- **Serwist** `^9.0.14` ‚Äî SW auto-generated to `public/sw.js` at build; disabled in dev
- **MDX** ‚Äî `gray-matter` + `react-markdown` + `remark-gfm` (no `@next/mdx`)
- **Analytics** ‚Äî `@vercel/analytics` + typed 12-event taxonomy in `lib/analytics.ts`
- **Deploy** ‚Äî Vercel; env var `NEXT_PUBLIC_SITE_URL` must be set

## Commands
```bash
npm run dev          # localhost:3000
npm run lint         # eslint . (flat config, next/core-web-vitals)
npm run typecheck    # tsc --noEmit
npm run build        # next build (production)
```

## CI gates (in order ‚Äî all must be green before merge)
1. `npm run lint`
2. `npm run typecheck`
3. `npm run build`
No test suite ‚Äî static site, low ROI. Add Vitest if interactive logic grows.

## Conventions
- **Styling**: reuse existing design tokens only (`@theme {}` in `globals.css`). No new colors, fonts, or inline styles.
- **Components**: reuse `Container`, `CTA`, `SectionReveal` before creating new ones.
- **Analytics**: fire typed events via `lib/analytics.ts`. Never call `window.gtag` directly.
- **Intake CTA**: use `site.intakeUrl` from `lib/site.ts` (driven by `NEXT_PUBLIC_TALLY_URL`). Never hardcode mailto.
- **MDX content**: `content/` directory. Case studies in `content/case-studies/`, FAQ in `content/faq/`.
- **Sitemap**: update `app/sitemap.ts` for every new route. New pages ‚â•0.7 priority.
- **Metadata**: every `page.tsx` must export a unique `<title>` and `<meta description>`.

## Key paths
```
app/                    # App Router pages and layouts
components/             # Shared React components
content/                # MDX source files
lib/analytics.ts        # Typed event taxonomy (12 events)
lib/site.ts             # Global site metadata + intakeUrl
app/globals.css         # Tailwind v4 @theme tokens (source of truth for design tokens)
app/sitemap.ts          # Sitemap ‚Äî update for every new route
public/sw.js            # Generated by Serwist at build (do not edit)
```

## Git / PR rules
- Branch protection: linear history, no force push, no direct push to main
- Always branch ‚Üí commit ‚Üí PR ‚Üí CI ‚Üí merge ‚Üí sync main
- PR title convention: `feat:`, `fix:`, `chore:`, `docs:`
- After merge: `git checkout main && git pull`

## Never do
- Push directly to main
- Add new npm dependencies without explicit instruction
- Use Tailwind v3 syntax (`theme.extend`, `@apply` with v3 utilities)
- Hardcode email/intake URLs ‚Äî use `lib/site.ts`
- Ship a PR with red CI
